# 🔍 User Research — Cognitive OS

## Research Objective

Understand how students and researchers currently organize knowledge, identify challenges with existing note-taking workflows, and validate whether AI-powered semantic search and knowledge visualization solve meaningful user problems.

---

# Research Questions

The study aimed to answer the following questions:

1. How do users currently store notes and learning material?
2. What difficulties do they face when retrieving information?
3. Which features do they value most in a knowledge management platform?
4. How do they currently search for previously saved information?
5. Would semantic search and knowledge graphs improve their workflow?

---

# Research Method

- One-on-one informal interviews
- Product walkthroughs using an interactive prototype
- Open-ended discussions

# Key Findings

## Pain Point 1

Users store notes across multiple platforms including notebooks, Google Docs, PDFs, and note-taking apps.

**Impact**

Information becomes fragmented and difficult to retrieve.

---

## Pain Point 2

Most users remember concepts instead of exact keywords.

Traditional keyword search often fails.

---

## Pain Point 3

Users spend significant time searching for previously saved information.

---

## Pain Point 4

Users struggle to understand relationships between different topics and notes.

---

# User Quotes

> "I know I saved it somewhere, but I don't remember where."

> "Sometimes I remember the idea but not the exact words."

> "My notes are everywhere."

---

# Insights

## Insight 1

Users value faster information retrieval more than advanced formatting features.

---

## Insight 2

Context-based search is more useful than keyword-based search.

---

## Insight 3

Visualizing relationships between notes helps users understand and revisit knowledge more effectively.

---

# Opportunity Areas

Based on the research, Cognitive OS should prioritize:

- AI-powered semantic search
- Intelligent note organization
- Interactive knowledge graph
- Tag-based categorization
- Simple and distraction-free user interface

---

# Design Decisions

Research directly influenced the MVP.

| User Need | Product Decision |
|-----------|------------------|
| Difficult to find notes | Semantic Search |
| Fragmented information | Centralized Dashboard |
| Poor organization | Tags |
| Unable to connect ideas | Knowledge Graph |

---

# Validation

Five participants interacted with the product prototype.

Most appreciated features:

- Semantic Search
- Knowledge Graph
- Clean Dashboard

Users suggested improving:

- Dashboard navigation
- Note organization
- Search result presentation

These suggestions were incorporated into subsequent iterations of the product.
